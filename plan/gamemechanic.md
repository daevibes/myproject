# 밸런스 및 메카닉 개선 기획서 (MVP 폴리싱)

본 문서는 Phase 5 구현 사항 중 실제 인게임 플레이 시 발생할 수 있는 밸런스 우려점 3가지에 대해, 시니어 게임 기획자/개발자 관점에서 게임성을 해치지 않으면서 디테일한 재미를 끌어올릴 수 있는 메카닉 개선안을 제안합니다.

---

## 1. 방어력(DEF) 절대 차감 공식의 한계와 개선안

### [현재 상황 분석]
현재 피해량 계산 공식은 `Math.max(1, 일반/보스 공격력 - 플레이어 총 방어력)` 입니다.
이 방식은 직관적이지만 초반에 운 좋게 방어력이 높은 영웅 등급 갑옷이나 방패를 획득할 경우, 거의 모든 적의 공격이 '1'로 들어오는 **절대 갓모드(Invincible state)**가 형성되어 생존 게임 특유의 긴장감이 사라집니다.

### [메카닉 개선 제안]
방어력 시스템을 **'절대 차감(Flat Damage Reduction)'** 에서 **'비율 감소(Percentage Mitigation)'** 모델로 변경하거나 최소 피해 보정을 도입해야 합니다.

*   **방안 A (추천): 롤(LoL)식 점근적 방어력 공식 도입**
    - 공식: `실제 피해량 = 적 공격력 * (100 / (100 + 플레이어 방어력))`
    - 효과: 방어력이 100일 때 데미지 50% 감소, 200일 때 66% 감소. 방어력이 아무리 높아도 데미지가 1이 되는 것을 방지하며, 스탯의 가치가 무한히 높아지는 것을 제어할 수 있습니다.
*   **방안 B: 최소 보장 피해량 (Minimum Damage Floor) 설정**
    - 공식: `Math.max(적 공격력 * 0.1, 적 공격력 - 플레이어 방어력)`
    - 효과: 현재의 직관적인 뺄셈 공식을 유지하되, 방어력이 아무리 높아도 **원래 공격력의 최소 10%**는 피해를 입도록 강제합니다. (예: 공격력이 20인 보스에게 맞으면 방어력이 100이어도 최소 2의 피해를 입음)

## 2. 몬스터 이동 스피드(Speed) 누적 스케일링 제한

### [현재 상황 분석]
웨이브마다 몬스터의 이동속도가 5%씩 증가합니다 (`WAVE_SPEED_SCALE = 0.05`).
초기 이속이 180인 달리는 적(Runner)의 경우 8웨이브 보스전 즈음에는 이속이 243을 돌파하여, 플레이어의 기본 이속(220)을 추월하게 됩니다. 카이팅(Hit & Run)이 핵심인 핵앤슬래시에서 적이 플레이어보다 상시로 빠르면 불쾌한 난이도 스파이크가 발생합니다.

### [메카닉 개선 제안]
속도 증가가 긴장감을 주는 것은 맞으나, 플레이어가 대처할 수 없는 "물리적 한계점"을 넘지 않도록 안전장치가 필요합니다.

*   **방안 A: 몬스터 이동속도 하드 캡(Max Cap) 설정**
    - 몬스터의 이동속도가 플레이어의 기본 이속(220)을 초과하지 못하도록 상한선을 두는 방법입니다.
    - 예: `스케일링 된 이속 = Math.min(210, 기본 이속 * 스케일)`
*   **방안 B: 플레이어 대시(Dash) 또는 피격 시 넉백 강화**
    - 몬스터가 빨라지는 것을 그대로 두되, 플레이어에게 위기 탈출 기믹을 줍니다. 
    - 평타 공격 성공 시 적들의 `KNOCKBACK_DURATION`(현재 80ms)을 늘려 카이팅할 수 있는 여유 프레임을 더 줍니다.
*   **방안 C: 점진적 스케일링 감소 (Soft Cap)**
    - 웨이브가 높아질수록 속도 증가폭이 5% -> 3% -> 1% 로 점차 줄어들게 설계하여 초반의 긴장감은 주되 후반 뇌절을 방지합니다.

## 3. 체력 유지력(Sustain) 수단의 부재

### [현재 상황 분석]
현재 유일한 체력 회복 수단은 8웨이브마다 등장하는 보스를 처치했을 때 발동하는 '여신의 가호(100% 회복)' 뿐입니다.
플레이어는 1~7웨이브 동안 몬스터들의 지속적인 칩 데미지(Chip Damage)를 포션이나 회복기 없이 버텨야 합니다. 이는 유저에게 과도한 스트레스로 작용할 수 있습니다.

### [메카닉 개선 제안]
전투 중 소소한 쾌감과 유지력을 제공하는 마이크로 보상(Micro Reward) 시스템이 필요합니다.

*   **방안 A: 체력 구슬 (HP Orb) 드롭 시스템 조미료 추가**
    - 몬스터 처치 시 3~5%의 낮은 확률로 인벤토리 슬롯을 차지하지 않고 **즉시 획득 및 사용되는 '체력 구슬(파란색/빨간색 작은 원)'**을 드롭힙니다.
    - 획득 시 최대 체력의 5% 정도를 즉각 회복해주어, 난전 속에서 구슬을 먹기 위해 위험을 무릅쓰는 다이내믹한 동선을 유도할 수 있습니다.
*   **방안 B: 아이템 옵션에 '생명력 흡수(Life Steal)' 특성 부여**
    - 특정 등급 이상의 무기나 특정 아이템(예: '뱀파이어의 검' 신규 추가)에 `lifeSteal: number` 스탯을 부여합니다.
    - 평타나 궁극기 타격 시 가한 데미지의 일정 비율(예: 3%)을 체력으로 회복하게 하여 공격 중심의 공격적인 플레이(Aggressive Play)를 장려합니다.
*   **방안 C: 웨이브 클리어 시 소량의 확정 회복**
    - 일반 웨이브(1~7)가 끝날 때마다 최대 체력의 10%를 고정으로 회복시켜 줍니다. "이번 웨이브만 넘기면 숨통이 트인다"는 확실한 단기 목표를 제공합니다.

## 확정 구현 스펙 (Phase 5 밸런스 패치)

아래 6건이 실제 코드에 반영됨.

### 1. DEF 최소 데미지 보정
- **공식**: `Math.max(Math.ceil(amount * 0.15), amount - defBonus)`
- **상수**: `MIN_DAMAGE_RATIO = 0.15` (`constants.ts`)
- Infantry ATK 5 → 최소 1, 보스 ATK 30+ → 최소 5

### 2. 몬스터 이속 Hard Cap
- **공식**: `Math.min(MONSTER_SPEED_SCALE_CAP, 1 + WAVE_SPEED_SCALE * (wave - 1))`
- **상수**: `MONSTER_SPEED_SCALE_CAP = 1.25`
- Runner 최대 = 180 × 1.25 = 225 px/s (플레이어 220보다 살짝 빠름)

### 3. 체력 구슬(HealOrb) 드롭
- 몬스터 처치 시 4% 확률 (`HEAL_ORB_DROP_CHANCE = 0.04`)
- maxHp의 5% 회복 (`HEAL_ORB_HEAL_RATIO = 0.05`)
- 초록색 원 (14px), 깜빡임 tween, 8초 후 소멸
- `game/entities/HealOrb.ts` 신규 엔티티

### 4. 아이템 드롭 확률 하향
- `DROP_CHANCE`: 0.40 → **0.10** (10마리당 1개)

### 5. 스폰 밀도 증가 + 대기 시간 단축
- `WAVE_REST_TIME`: 5000ms → **3000ms**
- 웨이브별 스폰 수 약 1.5배 상향 (Wave 1~3: 4~6/1~3, Wave 4: 5/3, Wave 5~7: 8~14/4~7, Wave 8 보스: 3/2)

### 6. 변경 파일 목록
| 파일 | 변경 내용 |

| `game/entities/Player.ts` | DEF 최소 데미지 공식 |
