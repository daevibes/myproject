## Phase 4 — 백엔드 연결

> 완료하면: 아이템 획득 시 Supabase DB에 저장됨. /inventory에서 확인. NFT 내보내기 가능.

**PO 사전 결정:** 없음 (API 이미 완성)

**개발:**
- [ ] 개발: 아이템 획득 시 `POST /api/inventory` 호출
- [ ] 개발: in_game_item_id가 드롭 테이블의 ID와 일치하는지 확인
- [ ] 개발: API 호출 실패 시 재시도 또는 로컬 큐잉
- [ ] 개발: 게임 화면 내 /inventory 이동 버튼
- [ ] 테스트: 게임에서 아이템 획득 → DB 저장 → /inventory에서 확인 → NFT 내보내기

**연결 대상 (이미 완성된 API):**
- `POST /api/inventory` — 아이템 저장
- `GET /api/inventory?game_user_id=xxx` — 아이템 조회
- `POST /api/fortem/mint/export` — NFT 내보내기

**PO 사전 결정 (Phase 3 끝나기 전에):**

- [ ] PO: 웨이브별 몬스터 수/속도/HP 증가율
- [ ] PO: 클리어 조건 (마지막 웨이브 생존? 보스 처치?)
- [ ] PO: 게임오버 조건 (공주 HP 0? 플레이어 HP 0?)

**개발:**
- [ ] 개발: 웨이브 매니저 (웨이브 번호, 몬스터 수 관리)
- [ ] 개발: 웨이브 간 대기 시간 (쉬는 시간)
- [ ] 개발: 난이도 곡선 (웨이브 올라갈수록 몬스터 강해짐)
- [ ] 개발: 게임오버 조건 처리 → 게임오버 화면

- [ ] 테스트: 웨이브 진행 → 난이도 상승 → 게임오버 또는 클리어

## Phase 의존 관계

```
Phase 1 (맵 + 이동)        ✅ 완료
    │
    ▼
Phase 2 (적 + 전투)        ✅ 완료
    │
    ▼
Phase 3 (아이템 + 로비)    ✅ 완료
    │
    ├──→ Phase 4 (백엔드 연결)  ← 아이템이 있어야 DB 저장 의미 있음
    │
    ▼
Phase 5 (게임 루프)        ← 전투 + 드롭이 있어야 웨이브 의미 있음
    │
    ▼
Phase 6 (UI/HUD)           ← 게임 루프가 있어야 표시할 정보가 있음
    │
    ▼
Phase 7 (에셋)             ← 모든 로직 완성 후 그림만 교체 (병렬 가능)
```
